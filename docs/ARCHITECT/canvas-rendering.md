# Canvas Rendering

> See also: ARCHITECTURE.md -- "Tech Stack > Canvas & Rendering", "Architecture Patterns > Canvas Rendering Architecture", "Component Architecture > Canvas Components"

## Overview

Stage uses a hybrid HTML/CSS rendering architecture for its canvas editor. Instead of relying on a single rendering technology, the system layers pure HTML/CSS elements captured via `modern-screenshot` (domToCanvas) for high-fidelity output. The editor preview is a live HTML DOM tree that doubles as the export source -- what you see is exactly what gets exported.

## Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Rendering engine | **HTML/CSS DOM (not Konva)** | Despite Konva being a dependency, the actual rendering uses pure HTML/CSS layers inside an `HTMLCanvasRenderer` container. This gives full CSS fidelity: gradients, blur filters, box-shadow, border-radius, 3D transforms, and blend modes all render natively without Canvas 2D API translation. |
| 3D transform capture | **modern-screenshot (domToCanvas)** | CSS 3D perspective transforms (`rotateX`, `rotateY`, `rotateZ`, `translateX/Y`, `scale`) cannot be rendered by html2canvas. `modern-screenshot` correctly captures `preserve-3d` and `perspective` CSS properties. |
| Layer compositing | **Stacked absolute-positioned divs** | Each layer (background, pattern, noise, main image, overlays, annotations) is an absolutely-positioned div inside the `HTMLCanvasRenderer` container, using `z-index` for ordering. The container uses `isolation: isolate` to create a stacking context. |
| Drag/resize interactions | **Native pointer events + manual coordinate math** | The main image and overlays use `pointerdown`/`pointermove`/`pointerup` listeners with manual offset calculations. Snap-to-center alignment guides appear when the image is within 6px of center. |
| Store synchronization | **EditorStoreSync component** | A null-rendering React component that synchronizes `useImageStore` (source of truth, tracked by undo/redo) into `useEditorStore` (rendering state consumed by canvas components). |

## Key Files

| File | Purpose |
|------|---------|
| `components/canvas/ClientCanvas.tsx` | Main canvas orchestrator. Dynamically imported (no SSR). Loads the primary image, manages selection state, keyboard shortcuts, and composes all HTML layers inside `HTMLCanvasRenderer`. Exposes `getCanvasContainer()` for export. |
| `components/canvas/EditorCanvas.tsx` | Wrapper component that shows `CleanUploadState` when no image is loaded, or `ClientCanvas` when an image exists. Also handles slide filmstrip and slideshow preview. |
| `components/canvas/EditorStoreSync.tsx` | Syncs `useImageStore` fields (image URL, scale, radius, background, frame, shadow, aspect ratio) into `useEditorStore`. Parses gradient strings for editor store format. |
| `components/canvas/html/HTMLCanvasRenderer.tsx` | The root container -- a `forwardRef` div with fixed dimensions, border-radius clipping, overflow hidden, and `isolation: isolate`. All layers are children of this div. |
| `components/canvas/html/HTMLBackgroundLayer.tsx` | Renders background (solid, gradient, image) with blur and noise support. Implements a crossfade transition (400ms) between background changes using A/B layer swap. |
| `components/canvas/html/HTMLMainImageLayer.tsx` | Renders the main uploaded image with frames (Safari, Chrome, arc, glass, polaroid), shadows (drop-shadow filter or box-shadow depending on frame type), CSS image filters, drag-to-reposition, corner resize handles, and rotation handle. |
| `components/canvas/html/HTMLPatternLayer.tsx` | Renders repeating pattern overlays (dots, grid, lines) generated by `generatePattern()`. |
| `components/canvas/html/HTMLNoiseLayer.tsx` | Renders a static noise texture image as a separate layer. |
| `components/canvas/html/HTMLTextOverlayLayer.tsx` | Renders text overlays with custom fonts, colors, shadows, and positioning. |
| `components/canvas/html/HTMLImageOverlayLayer.tsx` | Renders image overlays (front and back layers relative to the main image) with selection, drag, duplicate, and delete support. |
| `components/canvas/html/SVGAnnotationLayer.tsx` | SVG-based annotation layer for drawing arrows, rectangles, circles, and blur region boxes on top of the canvas. |
| `components/canvas/html/HTMLBlurRegionLayer.tsx` | Renders blur region rectangles using CSS `backdrop-filter: blur()`. These are applied as canvas post-processing during export since `backdrop-filter` doesn't render in domToCanvas. |
| `components/canvas/html/SnapAlignmentGuides.tsx` | Shows vertical/horizontal guide lines when the main image is dragged near center. |
| `components/canvas/overlays/Perspective3DOverlay.tsx` | Renders the main image with CSS 3D perspective transforms. Activates when any 3D property is non-default. Places a full-canvas perspective container at z-index 15 with `preserve-3d` transform style. |
| `components/canvas/frames/BrowserToolbar.tsx` | Safari and Chrome toolbar chrome rendered above the main image for browser frame types. |
| `components/canvas/frames/Frame3DOverlay.tsx` | Frame rendering for the 3D overlay path (same toolbar chrome, adapted for the 3D transform container). |
| `components/canvas/utils/canvas-dimensions.ts` | Pure function `calculateCanvasDimensions()` that computes all positioning: canvas size, image scaled size, frame offsets, padding, header heights, and center coordinates. Adapts padding ratio for mobile viewports. |
| `components/canvas/utils/shadow-utils.ts` | Shadow configuration types (`ShadowConfig`). |
| `components/canvas/utils/gradient-utils.ts` | Gradient parsing utilities. |
| `components/canvas/hooks/useImageLoading.ts` | `useBackgroundImage()` and `useOverlayImages()` hooks for async image loading with abort controller cleanup. Handles R2 URLs, blob URLs, and data URLs. |
| `types/canvas.ts` | Type definitions for templates, canvas objects (image, text), transforms, and export options. |
| `lib/canvas/templates.ts` | Predefined background templates (solid, gradient, shapes). |

## Data Flow

### Render cycle

```
useImageStore (source of truth)
  │
  ├─ EditorStoreSync ──→ useEditorStore (rendering state)
  │
  ▼
ClientCanvas (orchestrator)
  │
  ├─ calculateCanvasDimensions(image, viewport, canvas, screenshot, frame)
  │     → canvasW, canvasH, framedW, framedH, offsets, center coordinates
  │
  ├─ useBackgroundImage(backgroundConfig) → HTMLImageElement | null
  ├─ useOverlayImages(imageOverlays) → Record<id, HTMLImageElement>
  │
  └─ HTMLCanvasRenderer (container, ref → globalCanvasContainer for export)
       │
       ├─ z:0   HTMLBackgroundLayer (gradient/solid/image + blur + noise overlay)
       ├─ z:2   HTMLPatternLayer (repeating pattern)
       ├─ z:3   HTMLNoiseLayer (static noise texture)
       ├─ z:10  HTMLImageOverlayLayer[back] (overlays behind main image)
       ├─ z:10  HTMLMainImageLayer (image + frame + shadow + filters)
       │          OR
       ├─ z:15  Perspective3DOverlay (when 3D transforms active)
       ├─ z:16  3D Drag Layer (pointer capture for 3D position adjustment)
       ├─ z:20  HTMLTextOverlayLayer
       ├─ z:20  HTMLImageOverlayLayer[front]
       ├─ z:25  HTMLBlurRegionLayer (backdrop-filter blur regions)
       └─ z:30  SVGAnnotationLayer (arrows, shapes, blur-region drawing)
```

### Selection state

Selection is mutually exclusive across the main image, text overlays, image overlays, annotations, and blur regions. Clicking one type deselects all others. Clicking empty canvas area deselects everything. Interactions with editor panel controls (sliders, inputs, buttons) do not trigger deselection.

### 3D transform path

When any 3D property is non-default, the `HTMLMainImageLayer` is hidden and `Perspective3DOverlay` activates instead. This overlay:
1. Creates a full-canvas `perspective` container (`preserve-3d`)
2. Positions the framed image at the computed center
3. Applies a combined CSS transform: `translate(X%, Y%) scale(S) rotateX(Xdeg) rotateY(Ydeg) rotateZ(Zdeg)`
4. A transparent drag layer at z-index 16 captures pointer events for translate adjustment (3px drag threshold to distinguish clicks from drags)

## Trade-offs & Alternatives Considered

| Alternative | Why not chosen |
|-------------|---------------|
| **Pure Konva (Canvas 2D API)** | Cannot render CSS effects natively (gradients with oklch colors, backdrop-filter blur, complex box-shadows, 3D transforms). Would require reimplementing every CSS effect as Canvas 2D draw calls. The codebase previously used Konva but migrated to HTML/CSS for better fidelity. |
| **Pure DOM with html2canvas for export** | html2canvas cannot capture CSS 3D transforms (`perspective`, `rotateX/Y/Z`). `modern-screenshot` handles these correctly. html2canvas also has issues with oklch colors, complex filters, and font rendering. |
| **OffscreenCanvas for rendering** | OffscreenCanvas lacks DOM/CSS rendering -- it only provides Canvas 2D / WebGL APIs. The whole point of the HTML approach is leveraging the browser's CSS engine. |
| **react-moveable for all interactions** | Listed as a dependency but drag/resize is handled with native pointer events and manual math. This avoids the overhead of a large library for relatively simple position/scale/rotation logic and gives full control over snap-to-center behavior. |
| **Single store** | Two stores (image + editor) separate the undo/redo-tracked design state from transient rendering state. EditorStoreSync bridges them with a React effect. This avoids polluting the undo history with rendering-only changes. |
